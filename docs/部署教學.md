# 部署教學（約 15 分鐘，全程免費）

照著做，就能把 MWD 集點卡上線。不需要伺服器、不需要 POS、不需要 LINE。只要一個 Google 帳號。

---

## 一、建立資料庫（Google 試算表）

1. 到 [sheets.google.com](https://sheets.google.com) 新建一個空白試算表，命名例如「MWD 集點卡資料」。
2. 工作表會由程式自動建立，這步先放著就好。

## 二、貼上程式（Apps Script）

1. 在試算表上方點 **擴充功能 → Apps Script**。
2. 進入編輯器後，把預設的 `Code.gs` 內容**整段刪掉**，貼上本專案 `apps-script/Code.gs` 的全部內容。
3. 左側 **檔案 → ＋ → HTML**，命名為 **`Index`**（注意：不含 `.html`，大小寫要一致）。
   把本專案 `apps-script/Index.html` 的全部內容貼進去。
3b. 再新增一個 HTML 檔，命名 **`Admin`**，貼上 `apps-script/Admin.html` 的內容（店員查看今日通關碼的管理頁）。
4. （選用）點左上齒輪「專案設定」勾選「顯示 appsscript.json」，把 `apps-script/appsscript.json` 的內容貼上，可省去手動設定時區。
5. 按 **儲存**（Ctrl/Cmd + S）。

## 三、初始化資料表

1. 在編輯器上方函式下拉選 **`ensureSetup_`**，按 **執行**。
2. 第一次會跳出授權 → 選你的 Google 帳號 → 「進階 → 前往（不安全）」→ 允許。（因為是你自己的程式，安全。）
3. 回到試算表，會看到自動建立的 **Points** 與 **Config** 兩個工作表。

## 四、部署成網頁

1. 編輯器右上 **部署 → 新增部署作業**。
2. 類型選 **網頁應用程式（Web app）**。
3. 設定：
   - 執行身分：**我（你的帳號）**
   - 具有存取權的使用者：**任何人（Anyone）**
4. 按 **部署**，複製產生的 **網頁應用程式網址**（https://script.google.com/macros/.../exec）。
5. 用手機開這個網址測試：輸入手機 → 輸入通關碼 `1234`（預設）→ 應該會 +1 點。

## 五、做成 QR Code 貼櫃台

- 把上面的網址丟到任何免費 QR 產生器（例如 qrserver、QuickChart），印出來貼在櫃台或牆上。
- 或回來找我，我可以直接幫你把這個網址做成綠色品牌風的 A4 海報＋桌卡。

## 日常營運（出餐附碼流程）

本店為「桌上點餐＋送餐到桌」，且**平日消費滿 $180、假日滿 $280 才可集點**。為了把關金額、又不依賴櫃台互動，採用「**出餐附碼**」：今日通關碼不公開貼出，而是跟著「合格的那份餐」一起送到客人桌上。

### 每天早上（10 秒）
今日通關碼由**系統依日期自動產生**，不用再編輯試算表。
1. 用手機打開「店員管理頁」：把你的網頁網址後面加上 `?page=admin`（建議存成手機書籤），輸入管理 PIN（Config 的 `adminPin`，預設 `2468`，請改掉）。
2. 看到今天的通關碼後，把它寫到「今日通關碼卡」上（護貝小卡，可白板筆擦寫），準備一疊放出餐區。
   > ✅ 因為碼由系統算、店員只「照抄一次」到卡上，**系統與碼卡一定同步**，不會兩邊填到不同數字。
   > ⚠️ 別把碼貼在公開白板或牆上，否則未達最低消費的人也看得到。碼每天 00:00 自動換新。

### 不想手寫？直接列印碼卡（推薦）
店員管理頁有兩個列印按鈕，完全免手寫：

- **列印今日碼卡**：印出一頁多張「今天的碼」，裁開放出餐區，當天用、收工丟。每天早上印一次（約 10 秒）。
- **列印未來 30 天碼卡**：一次印整月，每天一張不同的碼，撕當天那張用——**一個月只印一次**，日常幾乎零動作。

兩種都不用手寫。護貝可擦寫的小卡（手寫版）則適合臨時、或當天沒印表機時備用。
> 進階：如果你的桌邊點餐／電子支付系統能在「訂單明細／收據」上印一段文字，也可以把當天的碼印在達標訂單的單據上，連碼卡都不用。

### 送餐時（店員）
- 送餐到桌時，**只在消費滿 $150 的餐點**托盤上放一張「今日通關碼卡」；未達標就不放。
- 這一步就同時完成了「確認有消費」＋「滿 $150」兩道把關，店員只多一個放卡動作。

### 客人自助（不用店員）
1. 掃桌上／卡上的集點 QR → 輸入手機號碼。
2. 輸入卡片上的今日通關碼 → 系統確認在店內（定位）後自動 +1。
3. 每支手機每天限 1 點。

### 兌換：用「店員兌換碼」
客人集滿 10 點會出現「兌換」按鈕，按下後要輸入 **staffCode**（預設 `8888`，在 Config 工作表，請改成只有店員知道的數字）。輸入正確才會扣點，避免客人自己亂兌換。

### 查資料
所有客人的點數都在 **Points** 工作表，可隨時看誰是熟客、誰快集滿。

### 為什麼這樣設計
- **滿 $150 把關**：碼只跟著合格餐點走，沒達標就拿不到碼。
- **不怕電子支付／桌邊點餐繞過櫃台**：給碼時機在「出餐」，每筆餐一定會經過。
- **防外流**：就算碼被拍照傳出去，遠端的人因為定位檢查（150 公尺）也蓋不了，且每支手機每天限 1 點。

## 常見調整（在 `Code.gs` 最上方 `CFG`）

| 參數 | 預設 | 說明 |
|---|---|---|
| `TARGET` | 10 | 集滿幾點可兌換 |
| `REWARD` | 指定餐點或飲料一份 | 獎勵內容文字 |
| `MIN_SPEND_WEEKDAY` | 180 | 平日（週二～週五）最低消費 |
| `MIN_SPEND_WEEKEND` | 280 | 假日（週六、週日）最低消費，系統自動依星期判斷 |
| `CODE_MODE` | auto | 'auto'＝系統依日期自動產生通關碼（推薦）；'manual'＝改用 Config 的 todayCode 自己設 |
| `REQUIRE_LOCATION` | false | false＝只記錄座標、不擋人；想正式封鎖遠端蓋章時改成 true |
| `SHOP_LAT` / `SHOP_LNG` | 25.0554725.. / 121.4315973.. | 店家經緯度 |
| `RADIUS_M` | 150 | 允許蓋章的範圍（公尺）。太小會誤擋、太大防護弱，建議 100–200 |
| 每天限蓋次數 | 1 | 在 `addStamp` 以「lastStampDate」控制，目前每支手機每天 1 次 |

> **Config 工作表的密碼類欄位**：`adminPin`（管理頁密碼）、`codeSalt`（自動碼的秘密種子，**請勿外流**；改掉它整月的碼都會跟著變）、`staffCode`（兌換碼）。請把預設值改成只有你知道的數字／字串。

> **定位（目前為記錄模式）**：`REQUIRE_LOCATION = false` 時，蓋章不因距離擋人，但每次都會把座標、與店家距離、是否在 `RADIUS_M` 內，記到 **Log** 工作表（欄位：time / phone / lat / lng / distM / within150m / pointsAfter）。觀察一段時間後，若發現有人從遠處蓋章，再把 `REQUIRE_LOCATION` 改成 `true` 即可正式封鎖。前端 `Index.html` 的 `CODE_LEN` 要與通關碼位數一致（自動碼為 4 位）。
> 之後若要「每天定時檢查」，可在 Apps Script 加一個時間觸發的函式去掃 Log 工作表（例如把距離 > 150m 的紀錄整理成日報寄到你信箱）。需要時再加。
> 注意：定位需要 HTTPS 才能運作，Apps Script 網址本身是 HTTPS，沒問題；但用 `file://` 直接開的單機試玩版不一定能取得 GPS，所以試玩版改用「在店內／在遠處」開關來模擬。

> 改完 `Code.gs` 要重新「部署 → 管理部署作業 → 編輯 → 版本選新版本 → 部署」才會生效。

## 安全與防作弊小結

- **加點**要正確的「今日通關碼」＋每支手機每天限一次 → 擋掉路人與重複灌點。
- **兌換**要「店員兌換碼」→ 客人無法自己歸零換獎。
- 想更嚴格可改成「每筆消費給不同的碼」，但會增加店員操作，視需要再調整。
